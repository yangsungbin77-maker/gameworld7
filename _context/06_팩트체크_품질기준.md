# 06. 팩트체크 · 품질 기준

픽셀은 "직접 파보는 게이머"라 신뢰가 생명이다. 페르소나로 글에 재미를 입히되, 사실은 절대 무너뜨리지 않는다.

## 사실/페르소나 분리 원칙 (핵심)
- **1인칭으로 입혀도 되는 것**: 감정, 찾아보며 헤맨 여정, "직접 정리해봤다", 게이머로서의 일반적 공감 프레임.
- **절대 날조 금지**: 사양·패치·출시일·가격·수치·기관명·인용. **안 해본 게임을 직접 플레이/클리어했다는 거짓 경험을 지어내지 않는다.**
- 잘 바뀌는 정보(가격·세일·할인율·사양·패치·출시일)는 단정 금지. "정가 기준", "세일 때 더 저렴", "○○년 기준", "공식 확인 권장"으로.

## 작성 전 의무
- 메인 키워드 **SERP 상위 분석**(상위 10개 중 5개+ 직접 열기) 후 작성 — 공통 필수항목은 빠짐없이 덮고, 콘텐츠 갭으로 정보 게인 각도 확보.
- 핵심 사실(게임 제목·장르·플랫폼·정책·수치)은 공식·신뢰 출처 또는 상위 글 2곳+로 교차확인. 출처 불명이면 "출처마다 다름"으로 처리하거나 뺀다.

## 발행 게이트 (다른 사이트와 동일 방향)
- 외부링크 1+·내부링크 1+·이미지가 본문에 실제로 있는지 `finalize.mjs` 단계 검증.
- 최소 분량·H2 수·사실 단정·과장·콜론(:) 마침 없는지 최종 점검.
- 헤르메스 심판 루프를 쓰는 경우 통과 기준 충족 후 발행.

## 리라이트 시 체크
- 원문의 사실·사양·정책·내부외부링크는 **보존**한다.
- 바꾸는 것은 톤·1인칭 framing·도입/마무리뿐. 팩트 라인은 건드리지 않는다.
- ⚠️ 1인칭을 입히다가 "내가 직접 깼다" 같은 거짓 경험이 끼어들지 않게 확인.

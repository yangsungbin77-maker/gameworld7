# 자동 발행 작업 지시서 (게임월드7)

이 문서는 예약 루틴이 **이틀에 한 번** 따라 실행하는 절차다. 목표는 단순 발행이 아니라, **그 분야를 잘 아는 사람이 직접 쓴 듯 깊이 있고 정확한 글 1편**을 내보내는 것이다. 각 글에 **외부링크 1개·내부링크 1개·이미지 1개**가 들어가고, **발행 전 반드시 웹으로 팩트체크**한다.

## 절차 (4단계)

작업 폴더는 저장소 루트(`gameworld7`)다. PATH에 node가 없으면 먼저 잡는다.

### 1단계 — 재료 받기
```
node automation/prepare.mjs
```
출력 JSON(= `automation/assignment.json`)에 이번 글 재료가 있다: `topic`, `pubDate`, `heroImage`, `internalLink{url,title}`, `externalLink{url,label}`.
"발행할 주제가 없습니다"로 멈추면 글을 쓰지 말고 **즉시 종료**한다(빈 글 금지).

### 2단계 — 리서치 (글 쓰기 전 필수)
`topic`(키워드)에 대해 **웹 검색(WebSearch/WebFetch)으로 최신·정확한 정보를 모은다.** 글에 쓸 구체적 사실 — 실제 게임 제목, 장르·플랫폼, 대략적 출시 시기, 주요 특징, 관련 정책 등 — 을 확인하고 근거를 기억해 둔다. 막연한 인상만으로 쓰지 않는다.

### 3단계 — 글 작성
`src/content/blog/<영문-slug>.md`를 작성한다. slug는 주제를 영어로 옮긴 짧은 kebab-case. 아래 **품질·형식 규칙**을 지킨다.

### 4단계 — 팩트체크 → 발행
작성한 글의 사실 주장을 **다시 한 번 검증**한다(아래 팩트체크 규칙). 확인 안 되는 내용은 고치거나 뺀다. 그다음:
```
node automation/finalize.mjs <위에서 정한 slug>
```
finalize가 링크·이미지·최소 분량·소제목 수를 검증하고, 빌드가 통과하면 커밋·푸시한다(Cloudflare 자동 배포). 검증/빌드 실패 시 메시지를 읽고 글을 보강해 다시 실행한다.

## 품질·형식 규칙 (반드시)

**깊이·구조 — 전문가처럼**
- 본문 **최소 1,800자**(한글 기준, 권장 2,000~2,800자). finalize가 너무 짧으면 발행을 막는다.
- `##` 소제목 **5~7개**. 각 섹션은 2~4문단으로 충실하게 채운다.
- 권장 흐름: ① 강한 도입(독자의 진짜 고민을 짚는다) → ② 핵심 개념·배경 → ③ 구체적 정보·실제 사례(게임 제목·설정·비교 등 구체적으로) → ④ 단계별 방법 또는 선택 기준 → ⑤ 흔한 실수와 해결 → ⑥ 마무리(요점 정리 + 한 줄 조언).
- **빈말 금지.** "재밌습니다", "좋아요" 같은 알맹이 없는 문장 대신 **구체적인 예시와 이유**를 쓴다. 실제 게임 이름·구체적 설정·비교를 들어 설명한다(단, 팩트체크 통과한 것만).
- 독자가 이 글 하나로 실제로 행동할 수 있을 만큼 친절하고 구체적이어야 한다.

**톤·표기**
- 존댓말 + 사람 냄새. 직접 해본 사람이 풀어주듯 쓴다. 정형 라벨 나열("특징/이런 사람에게") 금지.
- 한국어 문장을 콜론(`:`)으로 끝내지 않는다. `.`, `?`, `!`로 끝낸다(코드·키값 라벨 제외).

**필수 요소**
- 내부링크 1개: `[자연스러운 앵커](internalLink.url)` 형태로 문맥에 맞게 1회.
- 외부링크 1개: `[자연스러운 앵커](externalLink.url)` 형태로 1회.
- 이미지: frontmatter `heroImage`만 지정(본문에 따로 넣지 않는다 — 레이아웃이 상단 배너로 표시).
- frontmatter 형식:
  ```markdown
  ---
  title: '한국어 제목'
  description: '검색·미리보기에 노출될 한 줄 요약'
  pubDate: '<assignment의 pubDate 그대로>'
  heroImage: '<assignment의 heroImage 그대로>'
  ---
  ```

## 팩트체크 규칙 (발행 전 반드시)
- 글에 들어간 **모든 구체적 사실**(게임 제목·장르·플랫폼·출시 정보·가격·사양·정책·수치·인물)을 웹으로 확인한다.
- 확인되지 않으면 **삭제하거나** 단정하지 않는 표현으로 바꾼다("보통", "대체로", "~인 경우가 많다").
- **잘 바뀌는 정보는 단정하지 말 것.** 정확한 가격·할인율·구체적 날짜·세부 사양은 시점에 따라 달라지므로, 바뀌어도 맞는 표현으로 쓴다("정가 기준", "세일 때는 더 저렴" 등). "상시 ○○원" 같은 단정 금지.
- 출처가 불분명한 수치·"통계"는 쓰지 않는다.
- **의심스러우면 빼는 쪽을 택한다.** 틀린 정보를 발행하는 것이 정보가 조금 적은 것보다 나쁘다.

## 주제가 떨어졌을 때
prepare가 "발행할 주제가 없습니다"로 멈추면 발행하지 말고 종료한다(빈 글 금지). 주제는 사람이 `automation/topics.md`에 채운다.
